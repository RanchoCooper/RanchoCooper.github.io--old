什么是Hexo

Hexo 的作者是台湾大学生Tommy351, 基于 Node.js 开发, 是一个开源的静态博客生成器, 再配合上 Github Pages 的静态页面托管, 就能轻松管理个人博客了

但静态博客并非没有缺点. 动态博客更新文章时, 脚本是不变的, 每次提交只需要更新数据库即可.
而静态博客要重新生成文件, 也不支持增量式上传.

就算你是小白也无妨, 搭建好环境后, 只需要了解 Markdown 写法就能愉快地开始写作了

不过百度目前爬不了GitHub的数据, 如果想更好的发挥站图的作用，建议手动提交baidusitemap给百度, 详情见后续

指令概览

init

npm install hexo -g # 安装
npm update hexo -g  # 升级
hexo init <file>    # 初始化

常用

hexo n              # new       新建文章
hexo p              # public    发表文章
hexo r              # render    渲染文件
hexo g              # generate  生成文件
hexo s              # server    本地测试
hexo d              # deploy    部署站点
hexo clean          #           清理缓存

写作

hexo new page <title>
hexo new post <title>

服务器

hexo g --watch      # 监视文件变动
hexo s              # 会监视文件变动并自动更新, 无需重启服务器
hexo s -s           # 静态模式
hexo s -p 5000      # 更改端口号
hexo s -i x.x.x.x   # 自定义IP
hexo clean          # 清楚缓存

一些问题

找不到git部署

npm install hexo-deployer-git --save  # 需要这个包

hexo 3.0 部署类型不再是github，_config.yml中修改

type: git               # 不再是github
repository: git@user.github.com:user/user.github.io.git
branch: master

站点地图

通用插件

npm install hexo-generator-seo-friendly-sitemap --save

百度专属

npm install hexo-generator-baidu-sitemap@0.0.8 --save
# Hexo 3.0用这个
npm install hexo-generator-baidu-sitemap --save

生成地图

博客根目录/_config.yml中启用
plugins:
    hexo-generator-baidu-sitemap
    hexo-generator-sitemap

baidusitemap:
  path: baidusitemap.xml
sitemap:
  path: sitemap.xml

重新生成后根目录/public下就有了sitemap.xml和baidusitemap.xml

让百度收录博客

在百度站长平台 添加域名, 通过文件验证, 只要把验证文件放到public下, 然后再deploy

自动推送

那么, 如果我的站点更新了, 百度爬虫又是如何获取这些变更信息呢
接下来就是要提交链接, 把站点里的全部url告知给爬虫

让google收录博客

Google操作比较简单，就是向 Google站长工具 提交sitemap

登录Google账号，站点验证，选择站点，之后在 抓取——站点地图——添加/测试站点地图就好了
